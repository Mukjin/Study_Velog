<h2 id="📌-한줄-요약">📌 한줄 요약</h2>
<blockquote>
<p>컴퓨터가 데이터를 처리하는 근본적인 원리인 <strong>이진법(Base-2)</strong>과 메모리 단위(비트/바이트), 그리고 긴 이진수를 사람을 위해 압축하는 <strong>16진법</strong>에 대해 학습합니다.</p>
</blockquote>
<h2 id="🧐-배경--왜-배워야-하나">🧐 배경 &amp; 왜 배워야 하나</h2>
<ul>
<li><strong>이 개념이 필요한 이유</strong>: 인간은 손가락 10개를 기반으로 한 10진법에 익숙하지만, 컴퓨터는 전기가 통함(ON/1)과 통하지 않음(OFF/0)만을 인식하므로 철저히 2진법의 세계에서 동작합니다. </li>
<li><strong>해결하는 문제</strong>: 2진법과 16진법의 원리를 알면 메모리 공간이 한정된 상황에서 컴퓨터가 어떻게 양수와 음수를 나누어 저장하는지(2의 보수법) 근본적인 하드웨어 동작을 이해할 수 있어, 보다 깊이 있는 프로그래머로 성장할 수 있습니다.</li>
</ul>

<h2 id="📚-본문-정리">📚 본문 정리</h2>
<h3 id="1-10진수와-2진수의-차이">1. 10진수와 2진수의 차이</h3>
<ul>
<li><strong>10진수 (Base 10)</strong>: 0부터 9까지 10개의 기호를 사용합니다. 자릿수가 꽉 차면(9 다음) 상위 자릿수로 '1'을 올립니다.</li>
<li><strong>2진수 (Base 2)</strong>: 0과 1, 단 2개의 기호만 사용합니다. 기호가 부족하기 때문에 1 다음에는 곧바로 자릿수가 넘어가 <code>10</code>(일-빵)이 됩니다. <blockquote>
<p>💡 <strong>학습 꿀팁</strong>: 2진수의 동작 원리가 헷갈릴 때는, 항상 우리가 익숙한 10진수와 1:1로 비교해 가며 자릿수가 넘어가는 원리를 생각해보면 훨씬 이해가 빠릅니다.</p>
</blockquote>
</li>
</ul>
<h3 id="2-데이터의-단위-비트bit-바이트byte-워드word">2. 데이터의 단위: 비트(Bit), 바이트(Byte), 워드(Word)</h3>
<ul>
<li><strong>비트 (Bit)</strong>: 컴퓨터가 처리하는 가장 작은 단위. <code>0</code> 또는 <code>1</code> 단 하나의 값만 가질 수 있습니다.</li>
<li><strong>바이트 (Byte)</strong>: 8개의 비트가 모인 단위입니다. (<code>1 Byte = 8 Bit</code>)</li>
<li><strong>워드 (Word)</strong>: CPU가 한 번에 묶어서 처리할 수 있는 데이터의 양입니다. (예: 32비트 CPU에서는 한 번에 4바이트 묶음을 1워드로 취급합니다)</li>
<li><strong>표현 범위 계산</strong>: $n$개의 비트는 총 $2^n$개의 경우의 수를 표현할 수 있습니다. 예를 들어 4비트라면 총 16가지의 값을 가지며, 0부터 시작하므로 표현 범위는 <code>0 ~ 15</code>가 됩니다.</li>
</ul>
<h3 id="3-2진수를-압축하는-마법-16진법">3. 2진수를 압축하는 마법: 16진법</h3>
<p>2진수만 사용하면 숫자의 길이가 무한정 길어져 가독성이 몹시 떨어집니다. 이를 보완하기 위해 2진수를 <strong>뒤에서부터 4자리씩 끊어서</strong> 표기하는 16진법을 가장 널리 사용합니다.</p>
<ul>
<li>16진수는 0<del>9까지 숫자를 쓰고, 10부터 15까지는 부족한 숫자 기호 대신 **알파벳(A</del>F)**을 사용합니다.</li>
<li>예시: 2진수 <code>1100 1010</code>은 16진수로 <code>CA</code>로 압축됩니다. (1100 -&gt; C, 1010 -&gt; A)</li>
</ul>
<h3 id="4-컴퓨터가-음수를-표현하는-방법">4. 컴퓨터가 음수를 표현하는 방법</h3>
<p>컴퓨터의 메모리(2진수)에는 인간의 세계처럼 마이너스(<code>-</code>) 기호를 직접 쓸 수 있는 공간이 없습니다. 오로지 0과 1만 있습니다. 
그렇다면 16개의 표현이 가능한 4비트 환경에서 음수는 어떻게 처리할까요?</p>
<ul>
<li><strong>절반 나누기</strong>: 표현 가능한 경우의 수 16개 중 절반(8개)은 양수와 0을 위해 쓰고, 나머지 절반(8개)을 음수를 위해 쓰기로 약속했습니다.</li>
<li><strong>2의 보수법</strong>: 음수를 배치할 때 가장 효율적인 방식입니다. 복잡한 계산식의 장점 때문에 현대 컴퓨터는 음수를 표현할 때 절반의 공간을 활용하는 '2의 보수법'을 채택하여 사용합니다. (자세한 보수 변환법은 공식 문서나 교재의 상세 파트를 참고하시면 좋습니다.)</li>
</ul>
<hr />
<h2 id="💡-핵심-포인트-3가지">💡 핵심 포인트 3가지</h2>
<ol>
<li>컴퓨터는 전기적 신호의 한계로 인해 <strong>0과 1만을 사용하는 2진법</strong>으로 데이터를 다룹니다.</li>
<li>길고 복잡한 2진수를 사람이 쉽게 읽을 수 있도록 <strong>4자리씩 묶어서 알파벳과 함께 표기하는 것이 16진법</strong>입니다.</li>
<li>메모리에는 마이너스 부호를 직접 넣을 수 없으므로, 표현 범위의 <strong>절반을 나누어 '2의 보수법'이라는 체계로 음수를 대체 표현</strong>합니다.</li>
</ol>
