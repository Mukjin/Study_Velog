<h3 id="lombok-이란">Lombok 이란?</h3>
<p><img alt="" src="https://velog.velcdn.com/images/jinmuk4169/post/88c72950-be08-441c-8e75-8f10fa5bfb2e/image.jpg" /></p>
<ul>
<li><p>어노테이션 기반으로 코드를 자동완성 해주는 라이브러리이다.</p>
</li>
<li><p>Lombok을 이용하면 Getter, Setter, Equlas, ToString 등과 다양한 방면의 코드를 자동완성 시킬 수 있다.</p>
</li>
</ul>
<h3 id="gradle-이란">Gradle 이란?</h3>
<p><img alt="" src="https://velog.velcdn.com/images/jinmuk4169/post/a6b1207e-8364-4cfc-a584-ba81ff6d05eb/image.png" /></p>
<ul>
<li><p>Gradle은 Java, C/C++, Python 등과 같은 여러 언어를 지원하는 강력한 빌드 도구입니다. 자동화 빌드, 테스트, 배포, 출시 등의 프로세스를 관리하는 데 있어 도움을 줍니다.</p>
</li>
<li><p>Gradle은 Apache Ant와 Maven의 가장 좋은 기능을 결합하면서도, 유연성과 성능을 향상시킨 빌드 자동화 시스템입니다.</p>
</li>
</ul>
<h3 id="빌드도구는-다음의-반복적인-작업을-자동화한다">빌드도구는 다음의 반복적인 작업을 자동화한다.</h3>
<ul>
<li><p>필요한 라이브러리를 레포지토리(라이브러리가 저장되어 있는 곳)에서 다운로드한다.</p>
</li>
<li><p>소스코드를 컴파일한다.</p>
</li>
<li><p>테스트를 실행하여 보고서를 출력한다.</p>
</li>
<li><p>클래스 파일의 아카이브(여러파일이나 폴더를 하나로 정리하는 것)를 생성한다.</p>
</li>
<li><p>아카이브를 스테이징 환경(프로덕션 환경에 가까운 환경)등에 배포한다.</p>
</li>
</ul>
