<h4 id="프레임워크framework란">프레임워크(Framework)란?</h4>
<ul>
<li><p>틀(Frame) + work(일) 이라는 단어의 합성어이다. 일 구조, 혹은 작업 구조라는 뜻과 같은 프레임워크(Framework)는
어떠한 일을 처리하기 위한 구조를 제공한다. </p>
</li>
<li><p>쉽게 설명하면 소프트웨어나 애플리케이션 개발을 간단하게 해주는 뼈대이다.</p>
</li>
</ul>
<h4 id="프레임워크framework의-장단점">프레임워크(Framework)의 장단점</h4>
<ul>
<li><p>장점 - 프레임워크가 개발에 필요한 최소한의 기능을 제공하기 때문에 자신이 모든 기능을 작성할 필요가 없고, 애플리케이션 개발에 필요한 시간과 비용을 줄일 수 있다는 점이다.</p>
</li>
<li><p>단점 - 프레임워크를 사용한 개발에서는 프레임워크 고유의 사용법을 이해할 필요가 있다는 점이다.
ex)뼈대 부분이 제공하는 기능의 사용 방법을 이해해야 한다는 소리이다.</p>
</li>
</ul>
<h4 id="스프링-프레임워크spring-framework란">스프링 프레임워크(Spring Framework)란?</h4>
<ul>
<li><p>자바 개발 환경에서 사용되는 프레임 워크로 줄여서 스프링이라고도 한다.</p>
</li>
<li><p>스프링프레임워크에서는 개발을 편하게 할 수 있게 여러가지 기능을 제공하고 기능별로 프로젝트가 존재한다.</p>
</li>
</ul>
<p><img alt="" src="https://velog.velcdn.com/images/jinmuk4169/post/f99b3c9f-d87c-4212-8777-534d735937e3/image.png" /></p>
<h4 id="스프링-부트spring-boot">스프링 부트(Spring Boot)</h4>
<p>: 스프링 애플리케이션을 복잡한 설정 없이 빠르게 작성하는 기능을 제공한다.</p>
<h4 id="스프링-프로젝트spring-project">스프링 프로젝트(Spring Project)</h4>
<ul>
<li><p>스프링 MVC(Spring MVC)
: 웹 애플리케이션을 간단하게 생성하는 기능을 제공한다.</p>
</li>
<li><p>스프링 데이터(Spring Data)
: 데이터 접근에 관한 기능을 제공합니다.</p>
</li>
<li><p>스프링 배치(Spring Batch)
: 배치 처리 기능을 제공한다.</p>
</li>
<li><p>스프링 시큐리티(Spring Security)
:인증/허가 기능을 제공한다.</p>
</li>
</ul>
<h4 id="스프링-코어spring-core">스프링 코어(Spring Core)</h4>
<ul>
<li><p>스프링 DI(Spring DI)
: 의존성 주입(Dependency Injection) 기능을 제공한다.</p>
</li>
<li><p>스프링 AOP(Spring AOP)
: 관점 지향 프로그래밍(Aspect Oriented Programming)기능을 제공한다.</p>
</li>
</ul>
<p>여기서 다룰 기능은 스프링 부트(Spring Boot), 스프링 MVC(Spring MVC), 스프링 데이터(Spring Data), 스프링 DI(Spring DI), 스프링 AOP(Spring AOP)이다.</p>
