<h2 id="📚-본문-정리">📚 본문 정리</h2>
<h3 id="1-변수variable란-무엇인가">1. 변수(Variable)란 무엇인가?</h3>
<p>변수는 <strong>하나의 값을 저장하기 위한 메모리 공간</strong>을 의미합니다. 변수는 한 번에 여러 개의 값을 섞어서 저장할 수 없으며, 오직 <strong>하나의 값</strong>만 담을 수 있다는 중요한 특징이 있습니다.</p>
<h3 id="2-변수를-다루는-3가지-핵심-동작">2. 변수를 다루는 3가지 핵심 동작</h3>
<p>코딩 시 변수를 다루기 위해서는 다음 세 가지 동작만 정확히 기억하면 됩니다.</p>
<ul>
<li><p><strong>① 공간 만들기 (선언, Declaration)</strong>
가장 먼저 값을 담을 저장 공간을 컴퓨터 메모리에 만들어야 합니다. 어떤 타입의 데이터를 담을지 명시하고, 변수의 이름을 붙여줍니다.</p>
<pre><code class="language-java">int a; // 정수(int)를 담을 수 있는 저장 공간 'a'를 생성</code></pre>
</li>
<li><p>** ② 쓰기 (저장, Write) **
생성된 공간에 값을 집어넣습니다. 이때 <strong>대입 연산자(=)</strong>를 사용하며, 우측의 값을 좌측의 변수에 저장한다는 의미를 가집니다.</p>
<pre><code class="language-java">a = 10; // 변수 a에 숫자 10을 쓰기(저장)</code></pre>
</li>
<li><p>** ③ 읽기 (Read)** 
저장된 값을 프로그램에서 사용할 때는 단순히 변수의 이름만 적어주면 됩니다.</p>
<pre><code class="language-Java">System.out.println(a); // 변수 a에 저장된 값(10)을 읽어와 화면에 출력
</code></pre>
</li>
</ul>
<h3 id="3-변수의-값-변경-및-연산-원리">3. 변수의 값 변경 및 연산 원리</h3>
<p>프로그램이 실행되면서 변수의 값은 연산을 통해 얼마든지 바뀔 수 있습니다. 만약 변수에 새로운 값을 대입하게 되면, 기존에 있던 값은 완전히 지워지고 새로운 값으로 덮어씌워집니다.</p>
<pre><code class="language-java">a = a+1;</code></pre>
<p> 위와 같은 코드를 볼 때 수학적인 접근이 아닌 프로그래밍 관점으로 보아야 합니다. 대입 연산자(=)를 기준으로 우측의 수식이 먼저 계산됩니다.</p>
<ul>
<li><p>변수 a의 현재 값(예를 들어 10)을 읽어옵니다.</p>
</li>
<li><p>10에 1을 더해 11을 만듭니다.</p>
</li>
<li><p>계산된 결과 11을 다시 변수 a에 <strong>저장(쓰기)</strong>합니다.</p>
</li>
</ul>
<h3 id="4-두-변수의-값-교환하기-swapping">4. 두 변수의 값 교환하기 (Swapping)</h3>
<p>실제 알고리즘이나 비즈니스 로직을 작성하다 보면 두 변수의 값을 서로 맞바꿔야 하는 상황이 아주 빈번하게 발생합니다.</p>
<p>❌ 흔히 하는 실수 ❌</p>
<pre><code class="language-java">x = y;
y = x;</code></pre>
<p>위와 같이 직관적으로 작성하게 되면, 첫 번째 줄에서 x가 가진 원래 값이 y의 값으로 덮어씌워지면서 데이터가 완전히 유실됩니다. 결과적으로 두 변수 모두 똑같은 y의 값을 가지게 됩니다.</p>
<p>⭕ 올바른 접근법 ⭕ <strong>(임시 변수 활용)</strong>
콜라와 우유가 가득 담긴 두 컵의 내용물을 서로 바꾸려면, 음료를 잠시 부어둘 빈 컵이 하나 더 필요한 것과 같은 이치입니다. 이를 위해 임시 변수(tmp)를 선언하여 3단계를 거쳐 값을 옮겨야 합니다.</p>
<pre><code class="language-java">int x = 10;
int y = 20;
int tmp; // 빈 컵 역할을 할 임시 변수(Temporary Variable) 선언

tmp = x; // 1. x의 원래 값을 tmp에 임시 보관
x = y;   // 2. y의 값을 x에 안전하게 덮어쓰기
y = tmp; // 3. tmp에 보관해둔 원래 x의 값을 y에 최종 저장</code></pre>
<p>이 패턴을 적용해야 데이터 유실 없이 x와 y의 값이 성공적으로 교환됩니다.</p>
<h3 id="5-실습-꿀팁-intellij-단축키">5. 실습 꿀팁: IntelliJ 단축키</h3>
<p>강의의 실습 환경인 IntelliJ IDEA를 활용하면 타이핑 시간을 크게 단축할 수 있습니다. 변수의 값을 콘솔에 출력하여 디버깅하고 싶을 때 soutv를 입력하고 Tab 키를 누르면, System.out.println(&quot;변수명 = &quot; + 변수명); 코드가 자동으로 완성됩니다. 어떤 변수가 어떤 값을 담고 있는지 매우 직관적으로 파악할 수 있어 실무에서도 자주 쓰이는 팁입니다.</p>
<h3 id="💡-핵심-포인트-3가지">💡 핵심 포인트 3가지</h3>
<p>변수는 한 번에 하나의 값만 저장할 수 있는 메모리 상의 전용 공간입니다.</p>
<p>변수 활용은 <strong>선언(공간 생성) → 쓰기(데이터 할당) → 읽기(데이터 참조)</strong>라는 3단계 기본 프로세스로 이루어집니다.</p>
<p>두 변수의 값을 서로 교환할 때는 데이터 덮어쓰기로 인한 유실을 막기 위해 임시 변수(tmp)라는 '빈 컵'이 반드시 필요합니다.</p>
