<h4 id="1-의존성-주입-dependency-injection---di">1. 의존성 주입 (Dependency Injection -&gt; DI)</h4>
<ul>
<li><p>의존성(Dependency): 한 객체가 다른 객체를 필요로 하는 관계를 말한다. 예를 들어, 클래스 A가 클래스 B를 사용해야 한다면, 클래스 B는 클래스 A의 의존성이다.</p>
</li>
<li><p>주입(Injection): 객체가 필요한 의존성을 외부에서 제공해주는 것을 의미한다. 이는 생성자, 세터 메서드, 인터페이스 등을 통해 이루어질 수 있다.</p>
</li>
</ul>
<ul>
<li>클래스 의존 - 한 클래스가 다른 구체적인 클래스에 직접 의존하는 것을 의미한다. 예를 들어, 클래스 A가 클래스 B를 사용해야 할 때, 클래스 A는 클래스 B에 의존한다고 말한다. 이 의존성은 코드에서 명시적으로 나타나며, 변경이나 테스트에 어려움을 줄 수 있다.<pre><code>public class Engine {
  public void start() {
      System.out.println(&quot;Engine started&quot;);
  }
}
</code></pre></li>
</ul>

* 요약

1. 클래스 의존: 한 클래스가 다른 구체적인 클래스에 의존하는 것. 높은 결합도로 인해 코드 변경과 테스트가 어려울 수 있음.

2. 인터페이스 의존: 한 클래스가 인터페이스에 의존하고, 구체적인 구현은 다른 클래스가 담당하는 것. 낮은 결합도로 인해 유연성과 테스트 용이성이 높아짐.

#### 2. 관점 지향 프로그래밍 (Aspect oriented Programming -&gt; AOP)

- 중심적 관심사(Primary Concern) : 실현해야 할 기능을 나타내는 프로그램

- 횡단적 관심사(Crosscutting-Concerns): 본질적인 기능은 아니지만 품질이나 유지보수 등의 관점에서 반드시 필요한 기능을 나타내는 기능

- AOP를 간단히 설명하면 공통 처리 등의 '횡단적 관심사'를 추출하고 프로그램의 여러 곳에서 호출할 수 있게 설정함으로써 개발자는 실현해야 할 기능인 '중심적 관심사'에만 집중해서 작성하면 되는 구조이다.

#### AOP의 기본 개념

- Aspect(관점): 공통 기능(예: 로깅)을 모듈화한 것.

- Advice(조언): 특정 시점에 실행할 코드(예: 메서드 실행 전후에 로그를 남김).

- Join Point(조인 포인트): 조언이 적용될 수 있는 프로그램 내의 지점(예: 메서드 호출 시점).

- Pointcut(포인트컷): 특정 조인 포인트를 선별하는 조건(예: 특정 패키지의 모든 메서드).

- Weaving(위빙): 실제 코드에 Aspect를 적용하는 과정.
